HardSub Reader

Reads hardcoded (burned-in) video subtitles aloud in real time. Works with any video source: streaming sites, media players, video calls, or recorded lectures, in any language the OCR engine supports, including right-to-left scripts such as Arabic.

Requirements and installation

  • Windows 10 or 11. Windows 11 is strongly recommended: it provides the high-accuracy OneOCR engine.
  • Install the add-on from the .nvda-addon file and restart NVDA.
  • On Windows 11, the first time you start subtitle reading the add-on offers to set up the high-accuracy OneOCR engine. Answer Yes and approve the administrator prompt; the setup copies the engine files from your own Snipping Tool and reports when it is done. If you decline or the setup is unavailable, the add-on uses a lower-accuracy fallback engine and tells you so. The setup can also be run manually at any time with the setup_oneocr.ps1 script from the project page.

Advanced: if the bundled helper cannot run on your system, the add-on automatically falls back to running its helper script with a system Python installation (Python 3.10 or newer plus pip install pillow).

Usage

  • NVDA+alt+shift+s: toggle subtitle reading. A high beep confirms the OCR engine is ready.
  • Whether a new subtitle interrupts the one being spoken is controlled from the add-on settings; a keystroke for it can be assigned in the NVDA Input Gestures dialog, HardSub Reader category.
  • Keep the video window focused; the add-on scans the bottom portion of the foreground window.

The toggle keystroke can be reassigned in the NVDA Input Gestures dialog under the HardSub Reader category.

Settings

NVDA menu, Preferences, Settings, HardSub Reader:

  • Response speed: how quickly new subtitles are noticed. Faster settings use more CPU and battery.
  • Part of the video window to scan: subtitles usually sit at the bottom, so scanning only the bottom third is fastest and most reliable. Choose a larger area if a video shows text higher up.
  • Filtering of OCR misreads: the add-on can double-check each line across scans before speaking it, which prevents garbled text during subtitle fade animations at the cost of a small delay. Normal is recommended.
  • Do not repeat a line seen again within this many seconds: prevents the same subtitle being spoken twice when it flickers or stays on screen.
  • Stop reading the previous subtitle when a new one appears: keeps speech in sync with fast dialogue; turn it off if you prefer every line read to the end.
  • Subtitle language code: used only by the fallback engine on systems without OneOCR; the high-accuracy engine detects languages automatically.

Troubleshooting

  • A low beep means the OCR helper stopped; it restarts automatically. Repeated failures produce a spoken message.
  • The add-on keeps one diagnostic file on your computer: hardSubReader_sidecar.log in your temporary folder. It records only timestamps and technical error details for troubleshooting; it never contains subtitle text, screenshots, or anything about what you watched. It is capped at about one megabyte and can be safely deleted at any time.
  • If you hear a note about the legacy engine, run setup_oneocr.ps1 to enable the high-accuracy engine.
